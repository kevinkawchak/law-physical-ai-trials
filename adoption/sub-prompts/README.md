This folder is the output of **Process A**: from the single submitted prompt in
[`../prompts/prompt-adoption.md`](../prompts/prompt-adoption.md), Claude Code
generated four unique, comprehensive sub-prompts that **Process B** then runs in
sequence to grow the clinician confidence framework from scaffold to publication
quality. The method is adapted from the prior build's sub-prompts
([`../../review/sub-prompts/`](../../review/sub-prompts/)), re-targeted to the
`adoption/` tree and the eight confidence-question framework defined by the
prompt's CONFIDENCE GOALS. It is a new paper topic and does not rewrite the prior
review.

## Convention

Every sub-prompt opens with a `## prompt-<name>` heading followed by the prompt
text, mirroring the prior repository. Each stage that runs a sub-prompt writes a
paired `output-*.md` narrative file, and the three framework stages additionally
re-file their own `prompt-*.md` verbatim inside their directory.

## What is unique to this build (the adaptation)

1. **Target.** Every prompt targets `kevinkawchak/law-physical-ai-trials`,
   `adoption/`, and the clinician's decision to trust verified Physical AI at the
   bedside; repository release **v0.2.0**.
2. **Subject.** The objective is an evidence-paired confidence framework with
   eight body sections (competence, safety, transparency, oversight, equity,
   reliability, accountability, workflow), each pairing a bedside question with a
   cited fact.
3. **Media.** Colored Mermaid (native in Markdown, reproduced as colored TikZ in
   the compiled LaTeX) plus full-width tables and ASCII, all in the strict palette
   black, grayscales, `#EBCB8B`, `#D08770`, `#8B2E3F`. No raster images.
4. **Self-learning.** The draft leaves bracketed instructions naming exact source
   files; the full stage processes them into prose; the final stage applies the
   corrections identified from the full stage. The template
   (`../../review/template`) is reused as is and is not regenerated by a
   sub-prompt.

## Self-learning note

The sub-prompts encode the formatting lessons carried forward from the prior
build: the `>{\raggedright\arraybackslash}p{<width>}` table column convention, the
`\clearpage` / `\vspace` / `\hspace` page balancing, maximal widow and club
penalties so no line is stranded, the single-hyphen and section-symbol ("§")
rules, and the colored TikZ figure primitive that reproduces each Mermaid diagram
natively in LaTeX without a raster image.
